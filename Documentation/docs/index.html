MedApp Flutter App UI Kit

MedAPP UI Kit complete UI project using Flutter, it allows patients to book appointment from anywhere and anytime. also records userâ€™s medical data such as examination results, prescritopns and tests. it contains more than 26 screens ready to integrate with your backend (Firebase, PHP,Node.js ...) projects, you can reuse more than 35 widgets to customize your application.

The same code is used for both iOS and Android

Getting Started with UI Kit

  • After download and unzip the package, use preferred IDE (Android Studio / Visual Code / IntelliJ) to open the project medapp folder.

  • Click the Get dependencies or Packages get to install the libraries from pubspecs.yaml file.

  • Open the simulator to run iOS or Android

  • Then press the run button to start project

How to UI Kit?

In the <Your Poject>/lib folder you should find 6 subdirectories:

Components and Elements

In this folder <Your Poject>/lib/components there are +19 Widgets ready to use in your App:



Just you can call the widget class and customize it with you parameters and you can get the stylish widget integrated in your app


Example: 

//...
CustomButton(
    text: 'reschedule'.tr(),
    textSize: 14,
    onPressed: () {},
    padding: EdgeInsets.symmetric(
        vertical: 10,
        horizontal: 5,
    ),
    ),
//...

Model

In this folder <Your Poject>/lib/model you should find all models and Entities used in the App just you can link them with your backend (Firebase, PHP, Node.js ...)



Example:

Routing

<Your Poject>/lib/routes

The navigation mode in MedAPP UI Kit is route aspect it's easy and flexible approach 

//...
switch (settings.name) {
    case Routes.splash:
        return CupertinoPageRoute(builder: (_) => SplashPage());

    case Routes.login:
        return CupertinoPageRoute(builder: (_) => LoginPage());

    case Routes.signup:
        return CupertinoPageRoute(builder: (_) => SignupPage());

    case Routes.bookingStep2:
        return CupertinoPageRoute(builder: (_) => ChooseDoctorPage());
//...

Ready Pages

In this folder <Your Poject>/lib/pages there are +26 Pages/Screens pre-made:
Each screen is in a folder and have it's own widgets subfolder



Project Utils

In this folder <Your Poject>/lib/utils there are some configuration files for app theming, constants, colors, text styles ...etc.



i18n (internationalisation)

  • <Your Poject>/assets/languages folder contains internationalisation json files.

Build and Install App

  • After you making all you changes and customizations save the project.
  • Open Console, navigate to you project folder
  • Run flutter build apk --release you should get the apk file in the /output folder
  • Run flutter install to install you application on your connected devises
